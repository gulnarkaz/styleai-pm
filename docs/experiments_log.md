# Product Discovery & Lean Experiments

В основе разработки StyleGenie AI лежит методология **Lean Startup**: мы не строим догадок, а проверяем рискованные гипотезы через быстрые эксперименты.

## 1. Методология Discovery
Мы используем цикл **Build-Measure-Learn** для постоянной валидации продукта:
1. **Гипотеза:** Какую ценность мы хотим проверить?
2. **Эксперимент:** Минимально затратный способ проверки (MVP/Concierge/Smoke test).
3. **Метрика:** Как мы поймем, что гипотеза подтвердилась?
4. **Вывод:** Pivot (разворот) или Persevere (продолжаем).

| Гипотеза | Метод проверки (MVP) | Ключевая метрика | Результат | Решение |

## 3. Подход к MVP (Minimum Viable Product)
Чтобы не перегружать разработку, мы выделили **Core Value Proposition**:
* **Phase 0 (Manual):** Ручной подбор (Concierge MVP) для понимания логики стилистов.
* **Phase 1 (AI Prototype):** Базовый Vision AI + простая база товаров.
* **Phase 2 (Scaling):** Полноценная RAG-система и интеграция с брендами.

---

## 4. Итерационный процесс (Iteration Loop)
Каждые две недели мы проводим **Product Review**, где анализируем данные из `bias_monitor.py` и `ab_test_analyzer.py` для корректировки бэклога.
